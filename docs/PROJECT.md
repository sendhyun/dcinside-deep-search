## Issue: Chromium 다중 진입 URL 호환성

### 조치 결과

- `gall.dcinside.com`의 게시글 URL과 `enter.dcinside.com`의 목록 URL을 명시적 allowlist로 수용한다.
- URL 정책은 `src/dcinsideUrls.js`에서 구조적으로 검증하고, 검색 요청은 항상 `gall.dcinside.com`의 목록 URL로 canonicalize한다.
- sidePanel API가 없거나 실패하면 `sidepanel.html`을 확장 탭으로 연다.
- mini/mobile 갤러리는 관찰·테스트 전까지 지원하지 않는다.

### 검증 기준

- URL 정책 단위 테스트가 유사 도메인, 비HTTPS, custom port, credentials, unknown path를 거부한다.
- manifest 세 위치의 허용 host 목록이 정확히 일치한다.
- 기존 일반/마이너 목록 검색 회귀 테스트가 통과한다.

## Issue: 다중 검색어 결과 병합 충돌

- 상태: 수정 완료
- 심각도: 높음
- 영역: 다중 검색어 수집, 결과 병합, 결과 정렬
- 해결일: 2026-07-10

### 문제

다중 검색어 기능에서 두 번째 검색어의 검색 결과가 첫 번째 검색어의 검색 결과를 덮어씌우는 문제가 보고되었다. 쉼표 또는 줄바꿈으로 여러 검색어를 입력했을 때 각 검색어의 결과가 하나의 누적 목록으로 안정적으로 병합되지 않고, 검색어 간 결과 충돌이 발생할 수 있다.

### 영향

- 첫 번째 검색어 결과가 사라지거나 두 번째 검색어 결과로 대체되어 보일 수 있다.
- 검색어별 결과가 모두 보존되지 않아 수집 결과 신뢰도가 낮아진다.
- 중복 제거 및 `matchedKeywords` 병합 동작이 의도대로 유지되는지 재검증이 필요하다.
- 병합된 전체 결과가 최신순으로 정렬되지 않는 문제가 함께 보고되었다.

### 필요한 조치

- 다중 검색어별 수집 결과를 덮어쓰기 없이 누적 병합하도록 병합 경로를 점검한다.
- 같은 게시글은 하나의 row로 유지하되, 검색어 매칭 정보는 `matchedKeywords`에 누적한다.
- 여러 검색어의 결과를 검색어별 묶음이 아니라 전체 병합 결과 기준으로 `postId` 최신순 정렬한다.
- 쉼표 입력과 줄바꿈 입력 모두에 대해 회귀 테스트를 추가한다.

### 조치 결과

- `src/resultAggregation.js`에서 결과 병합과 표시용 최신순 정렬/제한을 분리했다.
- `background.js`에서 전체 누적 결과가 `maxResults`에 도달했다는 이유로 이후 검색어 수집 또는 병합을 막지 않도록 수정했다.
- 같은 게시글은 하나의 row로 유지하고, 여러 검색어에 걸린 경우 `matchedKeywords`에 발견 순서대로 누적되도록 유지했다.
- 최종 표시 및 저장 결과는 모든 검색어 결과를 병합한 뒤 `postId` 숫자 내림차순 최신순으로 정렬된 목록을 사용한다.
- `tests/resultAggregation.test.js`에 두 번째 검색어 결과가 병합 후보에 남고 최신순 표시 결과에 반영되는 회귀 테스트를 추가했다.
