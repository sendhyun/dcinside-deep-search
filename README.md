# DCInside Deep Search

> 이 프로그램은 개인용으로 바이브코딩을 통해 제작되었습니다. 프로그램 자체와 이 `README.md` 문서에는 오류가 있을 수 있으니, 사용 전 동작을 직접 확인하고 필요한 경우 코드와 안내 문구를 함께 점검해 주세요.

개인 사용용 Chrome Extension입니다. 디시인사이드 갤러리 내부 검색 결과 페이지나 갤러리 메인/목록 페이지에서 사용자가 직접 `수집 시작`을 누른 경우에만 결과를 수집하고, 현재 HTML에 실제로 존재하는 `다음 검색` 계열 href를 따라가며 오래된 검색 범위의 글 목록을 누적 표시합니다.

## 먼저 알아두세요

- Chrome, Edge, Brave, Opera, Whale 등 Manifest V3를 지원하는 최신 Chromium 브라우저에서 사용할 수 있습니다.
- Chrome 웹 스토어 설치 방식이 아니라, ZIP 파일을 받은 뒤 압축을 풀고 Chrome에 폴더를 직접 불러오는 방식입니다.
- 지원 대상은 `gall.dcinside.com`과 `enter.dcinside.com`의 PC 일반/마이너 갤러리 목록·검색 결과·게시글 진입 페이지입니다. 게시글 또는 `enter` 페이지에서 시작하면 수집은 안전한 `gall.dcinside.com` 목록 URL로 변환해 실행합니다.
- 브라우저가 sidePanel API를 지원하면 사이드 패널에서, 지원하지 않으면 동일 도구를 확장 탭에서 엽니다. mini/mobile 갤러리는 아직 지원하지 않습니다.
- 글 본문 전체를 저장하지 않고, 검색 결과 목록에 보이는 제목, 작성자, 날짜, 조회수, 추천수, 댓글수, 링크 같은 목록 정보만 취합합니다.
- 결과와 설정은 내 Chrome 확장 프로그램 저장공간에만 저장됩니다. 별도 서버로 보내지 않습니다.
- 디시인사이드 페이지 구조가 바뀌면 일부 기능이 동작하지 않을 수 있습니다.
- 너무 짧은 간격으로 많이 수집하면 디시인사이드 요청 제한이나 CAPTCHA가 나올 수 있습니다.

## 처음 설치하는 분을 위한 안내

이 확장 프로그램은 Chrome 웹 스토어에서 설치하는 방식이 아니라, GitHub에서 ZIP 파일을 받은 뒤 Chrome에 직접 불러오는 방식입니다.

### 1. GitHub에서 ZIP 파일 받기

1. 이 프로젝트의 GitHub 페이지를 엽니다.
2. 초록색 `Code` 버튼을 누릅니다.
3. `Download ZIP`을 누릅니다.
4. 다운로드가 끝날 때까지 기다립니다.

보통 ZIP 파일은 Windows의 `다운로드` 폴더에 저장됩니다.

### 2. ZIP 파일 압축 풀기

1. 다운로드한 ZIP 파일을 마우스 오른쪽 버튼으로 클릭합니다.
2. `압축 풀기`, `Extract All...`, 또는 `모두 추출`을 누릅니다.
3. 압축을 풀 위치를 정한 뒤 `추출`을 누릅니다.

중요: Chrome에는 ZIP 파일 자체가 아니라 압축을 푼 폴더를 넣어야 합니다.

압축을 풀면 폴더가 생깁니다. 이 폴더 안쪽에 `dcinside-deep-search` 폴더가 있고, 그 안에 `manifest.json` 파일이 있어야 합니다.

### 3. Chrome에 확장 프로그램 불러오기

1. Chrome을 엽니다.
2. 주소창에 `chrome://extensions`를 입력하고 Enter를 누릅니다.
3. 오른쪽 위의 `개발자 모드`를 켭니다.
4. `압축해제된 확장 프로그램을 로드`를 누릅니다.
5. 압축을 푼 폴더 안에서 `manifest.json` 파일이 들어 있는 폴더를 선택합니다.

이 저장소를 통째로 ZIP으로 받은 경우, 보통 `dcinside-deep-search` 폴더를 선택하면 됩니다.

정상적으로 설치되면 Chrome 확장 프로그램 목록에 `DCInside Deep Search`가 나타납니다.

자주 사용할 경우 Chrome 오른쪽 위의 퍼즐 모양 아이콘을 누른 뒤 `DCInside Deep Search` 옆의 고정 아이콘을 눌러 상단에 고정해두면 편합니다.

### 4. 디시인사이드에서 사용하기

1. 디시인사이드에서 원하는 갤러리에 들어갑니다.
2. 검색 결과 페이지에서 시작하려면 갤러리 안에서 찾고 싶은 단어를 먼저 검색합니다. 갤러리 메인에서 시작하려면 검색하지 않은 목록 페이지 그대로 둡니다.
3. 해당 페이지가 열린 상태에서 Chrome 오른쪽 위의 퍼즐 모양 아이콘을 누릅니다.
4. `DCInside Deep Search`를 누릅니다.
5. 팝업에서 `사이드 패널 열기`를 누릅니다.
6. 갤러리 메인에서 시작하는 경우 사이드 패널의 `수집할 검색어`에 검색어를 입력합니다.
7. 오른쪽 사이드 패널에서 `수집 시작`을 누릅니다.

수집이 시작되면 `취합된 글 목록`에 검색 결과가 쌓입니다. 목록의 글을 클릭하면 원문 글이 새 탭으로 열립니다.

여러 검색어는 줄바꿈 또는 쉼표로 구분할 수 있습니다. 수집은 검색어별로 순차 진행되지만, 표시 목록은 검색어별 묶음이 아니라 취합 결과 기준 최신순으로 정렬됩니다. 검색어가 많으면 시간이 오래 걸리고 요청 제한이나 CAPTCHA가 발생할 수 있습니다.

검색 결과 페이지에서 시작하면 현재 페이지의 검색어가 자동으로 입력될 수 있습니다. 갤러리 메인/목록 페이지에서 시작하면 `수집할 검색어` 칸에 직접 입력해야 합니다.

여러 검색어 예시:

```text
검색어A
검색어B
검색어C
```

쉼표로도 입력할 수 있습니다.

```text
검색어A, 검색어B, 검색어C
```

수집 중에는 현재 페이지와 다음 검색 범위를 차례로 확인합니다. 확장 프로그램은 페이지 번호를 임의로 계산하지 않고, 현재 HTML에서 찾은 실제 `다음 검색` 링크만 따라갑니다.

## 사이드 패널 구획별 기능

### 검색 컨텍스트

현재 페이지에서 확장 프로그램이 인식한 정보를 보여줍니다.

- `갤러리`: 현재 갤러리 이름 또는 갤러리 ID입니다.
- `검색어`: 현재 검색 결과 페이지에서 감지한 검색어입니다. 갤러리 메인에서는 `입력 필요`로 보일 수 있습니다.
- `검색 타입`: 제목+내용 검색 같은 디시인사이드 검색 방식 값입니다.
- `시작 URL`: 수집을 시작할 기준 URL입니다.

여기 값이 `unknown`으로만 보이거나 지원 페이지가 아니라고 나오면, PC 디시인사이드 갤러리 목록/검색 결과 페이지인지 확인하세요.

### 수집할 검색어

갤러리 메인/목록 페이지에서 검색을 시작할 때 사용하는 입력칸입니다.

- 한 개 검색어만 입력해도 됩니다.
- 여러 검색어는 줄바꿈 또는 쉼표로 구분합니다.
- 같은 검색어를 중복 입력하면 한 번만 사용합니다.
- 검색 결과 페이지에서 시작할 때는 감지된 검색어가 자동 입력될 수 있습니다.

### 수집 버튼

- `수집 시작`: 현재 페이지를 진단한 뒤 수집을 시작합니다.
- `중지`: 현재 수집을 멈춥니다.
- `목록 비우기`: 저장된 취합 목록과 최근 상태를 삭제합니다.

### 취합된 글 목록

수집된 글 목록을 보여줍니다.

- 글 제목을 누르면 원문 글이 새 탭으로 열립니다.
- 각 항목에는 작성자, 날짜, 조회수, 추천수, 댓글수, 수집 단계가 표시됩니다.
- 여러 검색어에 걸린 같은 글은 한 줄만 남기고, `검색어 A, B`처럼 매칭된 검색어 정보가 함께 표시됩니다.
- 여러 검색어 결과는 전체 취합 결과 기준으로 최신순 정렬됩니다. 글 번호인 `postId`를 숫자로 비교해 큰 번호가 위에 오도록 정렬합니다.
- 글 번호를 읽을 수 없는 항목이 있어도 목록 표시가 중단되지 않도록 기존 상대 순서를 최대한 유지합니다.

### 진행 상태

현재 수집이 어디까지 진행됐는지 보여줍니다.

- `단계`: 검색 범위를 몇 단계까지 이동했는지 나타냅니다.
- `방문 URL`: 수집 중 방문한 URL 수입니다.
- `결과`: 현재 취합된 결과 수입니다.
- `마지막 오류`: 마지막으로 발생한 오류 메시지입니다. 없으면 `없음`으로 표시됩니다.
- `마지막 요청`: 마지막으로 요청한 URL입니다.
- `마지막 next`: 다음 검색 범위로 이동할 때 사용한 마지막 URL입니다.

### 목록 필터

이미 취합된 목록을 화면에서 좁혀 볼 때 사용합니다. 원본 저장 결과를 지우는 기능은 아닙니다.

- `제목`: 제목에 특정 단어가 들어간 글만 표시합니다.
- `작성자`: 작성자 이름으로 표시 목록을 좁힙니다.
- `최소 추천 수`: 입력한 추천 수 이상인 글만 표시합니다.

필터를 지우면 다시 전체 취합 목록이 보입니다.

### 수집 설정

처음 사용하는 경우 기본값 그대로 쓰는 것을 권장합니다.

- `최대 수집 글 수`: 화면에 유지할 최대 결과 수입니다. 기본값은 30개이고, 1에서 100 사이로 설정됩니다.
- `요청 간격 ms`: 다음 페이지 요청 전 기다리는 시간입니다. 기본값은 300ms입니다.
- `중복 제거`: 같은 글 번호나 같은 URL이 여러 번 나오면 한 번만 표시합니다.
- `불확실한 다음 링크 자동 이동 허용`: 다음 검색 링크 후보가 확실하지 않아도 첫 후보를 따라갑니다.

요청 제한이나 CAPTCHA가 걱정되면 `요청 간격 ms`를 `1200`, `2000`, `3000`처럼 더 크게 늘리세요. `불확실한 다음 링크 자동 이동 허용`은 일반적으로 꺼두는 것을 권장합니다.

## 다중 검색어 동작

다중 검색어는 편의를 위한 기능이지만, 요청 자체는 병렬로 보내지 않습니다.

1. 첫 번째 검색어를 수집합니다.
2. 첫 번째 검색어가 끝나면 두 번째 검색어를 수집합니다.
3. 이후 검색어도 같은 방식으로 순차 처리합니다.
4. 표시할 때는 모든 검색어 결과를 하나로 병합합니다.
5. 같은 글은 한 줄만 남기고 `matchedKeywords`에 매칭된 검색어를 발견 순서대로 누적합니다.
6. 병합된 전체 목록을 최신순으로 정렬해 보여줍니다.

따라서 검색어가 `A, B`일 때 결과창은 `A 전체 다음 B 전체` 순서가 아니라, A와 B 결과를 합친 뒤 글 번호 기준 최신순으로 보입니다.

### 자주 막히는 부분

- 확장 프로그램을 로드할 수 없으면 ZIP 파일을 그대로 선택한 것은 아닌지 확인하세요.
- 선택한 폴더 안에는 반드시 `manifest.json` 파일이 있어야 합니다.
- Chrome 상단에 확장 프로그램이 안 보이면 퍼즐 모양 아이콘을 눌러 `DCInside Deep Search`를 찾으세요.
- `검색 결과 페이지로 인식하지 못했습니다`가 나오면 디시인사이드 갤러리 안에서 검색한 결과 페이지인지 확인하세요.
- 요청 차단이나 CAPTCHA가 나오면 수집을 멈추고, `수집 설정`에서 요청 간격을 더 크게 늘려 사용하세요.

## 구현 요약

- Manifest V3 기반 확장 프로그램입니다.
- 사이드 패널에서 수집, 중지, 목록 비우기를 실행합니다.
- 수집 시작 시 현재 페이지 진단을 자동으로 먼저 실행합니다.
- 검색 결과 HTML은 저장하거나 재사용하지 않고 매 수집마다 다시 요청합니다.
- `src/parser.js`는 2026-07-08에 직접 관찰한 구조를 `OBSERVED_RULES`로 분리했습니다.
- `page` 번호 증가 크롤링은 구현하지 않았습니다. 다음 이동은 `findNextSearchRangeUrl(html, currentUrl)`가 현재 HTML에서 찾은 실제 href만 사용합니다.
- `search_pos`가 10000 단위로 바뀌는 패턴은 후보 검증 근거로 기록하지만, 다음 URL을 직접 계산해서 만들지는 않습니다.
- 확신이 낮거나 후보가 여러 개면 기본 설정에서 자동 이동하지 않습니다.

## Discovery 결과

자세한 내용은 `docs/DISCOVERY.md`에 있습니다.

직접 관찰한 핵심 구조:

- 갤러리 id: URL `id` 값.
- 검색어: `#s_keyword`.
- 검색 타입: `#s_type`.
- 검색 범위 이동 후보: `a.search_next[href]`.
- 검색 범위 이동 파라미터: 관찰된 샘플에서 `search_pos`.
- 결과 테이블: `.gall_listwrap.list table.gall_list`.
- 게시글 행: `tr.ub-content.us-post[data-no]`.
- 제목 링크: `.gall_tit a[href*="/board/view/"]:not(.reply_numbox)`.

접근이 불가능한 환경에서는 `tools/dc-discovery-snippet.js`를 현재 검색 결과 페이지의 브라우저 콘솔에 붙여넣어 문제 해결용 구조 정보를 확인할 수 있습니다.

## 설치 방법

1. Chrome에서 `chrome://extensions`를 엽니다.
2. 개발자 모드를 켭니다.
3. `압축해제된 확장 프로그램을 로드`를 누릅니다.
4. 이 폴더의 `dcinside-deep-search` 디렉터리를 선택합니다.

## 사용 방법

1. 디시인사이드 특정 갤러리 검색 결과 페이지 또는 갤러리 메인/목록 페이지에 들어갑니다.
2. 확장 프로그램 사이드 패널을 엽니다.
3. 갤러리 메인에서 시작하는 경우 `수집할 검색어`에 검색어를 입력합니다. 여러 검색어는 줄바꿈 또는 쉼표로 구분합니다.
4. `수집 시작`을 누릅니다.
5. 진행 상태와 결과를 사이드 패널에서 확인합니다.
6. 취합된 글 한 칸을 클릭하면 원문 글로 이동합니다.

## 진단 모드

진단 모드는 현재 페이지에서 검색어, 결과 영역, 샘플 결과, 다음 검색범위 후보를 확인합니다. 기본 사용 흐름에서는 글 목록 취합만 사용하면 됩니다.

## 문제가 생겼을 때 확인할 항목

- `지원 가능한 디시인사이드 갤러리 페이지가 아닙니다`: 현재 페이지가 PC 일반 갤러리 또는 마이너 갤러리 목록 페이지인지 확인합니다.
- `갤러리 메인에서는 수집할 검색어를 입력하세요`: 검색하지 않은 갤러리 메인/목록 페이지에서는 사이드 패널 검색어 입력칸에 검색어를 입력해야 합니다.
- `다음 검색범위 링크 후보가 불확실합니다`: 후보가 여러 개거나 검색 조건 보존 근거가 부족합니다. 페이지 하단의 `다음 검색` 링크가 실제로 있는지 확인합니다.
- `요청 차단 가능성` 또는 `CAPTCHA 가능성`: 수집을 멈추고 요청 간격을 늘립니다.
- `Resource::kQuotaBytes quota exceeded`: 확장 저장공간이 찼다는 뜻입니다. `목록 비우기`로 저장된 취합 목록과 최근 상태를 지운 뒤 다시 수집합니다.
- 결과가 비어 있음: 실제 검색 결과가 없는 범위일 수 있습니다. `DISCOVERY.md`의 결과 0개 상태와 비교합니다.

## 구조 변경 시 수정 파일

- `src/parser.js`: observed selectors, 결과 파싱, next URL 확신도 규칙.
- `src/discovery.js`: 문제 해결용 페이지 구조 수집 항목.
- `docs/DISCOVERY.md`: 새로 관찰한 구조와 근거.
- `tests/parser.test.js`: 새 구조 기반 fixture.

## 한계와 주의사항

- 2026-07-08에 관찰한 일반 PC 갤러리 구조를 기준으로 했습니다.
- 일반 갤러리와 마이너 갤러리의 PC 목록 URL을 우선 지원합니다. 미니 갤러리와 모바일 URL은 별도 확정하지 않았습니다.
- CAPTCHA와 403/429 페이지는 직접 관찰하지 못해 보수적인 문자열 감지와 HTTP 상태 감지만 넣었습니다.
- 개인용 도구이며 외부 서버로 결과를 보내지 않습니다.
- 기본 요청 간격은 300ms입니다. 더 안전하게 쓰려면 값을 늘리세요.
